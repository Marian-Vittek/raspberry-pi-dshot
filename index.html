Raspberry Pi Dshot protocol implementation.

Implementation of DSHOT protocol for Raspberry Pi. It allows to control ESCs directly via the GPIO pins.

It uses busy wait and 'clock_gettime' for timing. It handles DSHOT 150 protocol well. Other versions may cause problems.

The code is inspired by dmrlawson/raspberrypi-dshot. Compared to the original, this version differs in two points:
  • It uses "clock_gettime" for timing instead of an empty loop.
  • It allows multiple frames to be sent to multiple pins at once.
To get started download source code from github.

To compile and run an example spinning 4 motors type: 
    make test
    sudo chrt -f 99 ./test
You may also need to edit 'motor-dshot.c' and adjust DSHOT_AD_HOC_OFFSET to your ESC.
